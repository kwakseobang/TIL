# 메모리 구조
- 프로그램이 실행되면 운영체제는 프로그램이 실행되기 위해 다양한 메모리 공간을 할당한다.

1. 코드 영역
2. 데이터 영역
3. 힙 영역
4. 스택 영역


- 코드(code) 영역
  - 실행할 프로그램의 코드가 저장되는 영역으로 텍스트 영역이라고도 불린다.
  - 코드가 기계어로 저장
  - 프로그램 시작~종료까지 메모리에 남는다.
- 데이터(data) 영역
    - 프로그램의 전역변수와 정적변수가 저장되는 영역이다.
    - 프로그램 시작과 함께 할당되며, 종료 시 메모리 해제
- 힙(heap) 영역
  - 사용자가 직접 공간을 할당/해제 할수 있는 영역
  - 힙 영역은 사용자에 의해 메모리 공간이 동적으로 할당되고 해제됨.
  - FIFO 이다.
  - 참조 타입의 값이 힙 영역에 할당된다
  - 런타임 시 크기가 결정된다.
  - 메모리가 낮은주소 -> 높은주소로 방향으로 메모리 할당
- 스택(Stack) 영역
  - 임시 메모리 영역이다
  - 함수 호출 시 함수의 지역변수, 매개변수, 리턴 값 등이 저장되고 함수가 종료되면 저장된 메모리 공간도 해제된다.
  - LIFO
  - 메모리가 높은주소 -> 낮은주소로 방향으로 메모리 할당
  -  컴파일 타임에 크기가 결정됨
  


### 런타임, 컴파일 타임
- 런타임
    -  컴퓨터 프로그램이 실행되고 있는 동안의 시간 또는 상태
- 컴파일 타임
  - 사용자가 작성한 코드가 컴파일을 진행하는 과정
