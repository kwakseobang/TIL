
# Text Processing

- Structured data
  - 일정한 규격이나 형태를 지닌 숫자 데이터
- Unstructured data
  - 그림, 영상, 문서 등과 같이 형태와 구조가 다른 구조화되지 않은 데이터

### 자연어 코퍼스 (Corpus)

- Corpus
    - 말뭉치
    - 매우 많은 수의 문장을 정제하여 모아둔 것
    - “대량의 텍스트 데이터”
    - 자연어 연구를 위해 특정한 목적을 가지고 언어의 표본을 추출한 집합
    - 통계/딥러닝 기반 자연어 처리에서 가장 핵심을 담당하는 자료
    - 컴퓨터에 저장된, 쓰이거나 말해진 자연어 자료 모음
    - 둘 이상의 코퍼스: corpora

- 자연어 코퍼스의 기본적인 조건
  - 풀고 싶은 자연어처리 작업의 특성을 잘 담아 낼 수 있도록 다양한 패턴의 데이터를 포함
  -  데이터가 유의미한 규모로 확보되어야 하고, 그 데이터가 대표성을 지녀야 함

### Tokenization (Chunking)
 일반적인 전처리 (Preprocessing) 과정
- 데이터 수집 과정에서 얻어낸 코퍼스 데이터가 필요에 맞게 처리되지 않은 상태라면, 해당 데이터를 용도에 맞게 전처리하는 작업이 필요
- 원시 데이터(Raw data)로부터 유용한 데이터 문자열로 변환하는 간단한 프로세스
- 단락과 문장을 보다 쉽게 의미를 할당할 수 있는 더 작은 단위로 활용
**주어진 코퍼스 내 자연어 문장들을 “Token”이라는 최소 단위로 나누는 작업**
보통 띄어쓰기로 나누는 경우가 많음
- Token
   - 자연어 내에서 의미를 가지는 최소 단위



### How to tokenize (Chunk)

- Sentence Tokenization (Chunking)
  - 토큰의 단위가 문장 (Sentence)인 토큰화 방법
  - 문장부호 (“.” “!” “?” 등)를 기준으로 코퍼스를 분리
  - 예외 존재
    - 문장 중간에 . 등 문장 부호가 있을 때. 예외처리를 해줘야함 커스텀해서 만져줘야댐

- Word Tokenization (Chunking)
  - 보편적인 토큰화 방법
  - 특정 구분기호를 가지고 텍스트를 나누게 되며, 기본적으로는 **공백을 구분자로 사용**


### Tokenization (Chunking) 시 고려사항 in LLM
1. Context 의 길이
   -  기사, 책과 같은 긴 문서 VS 트윗이나 메시지와 같은 짧은 콘텐츠
   -  **중요한 파트**
2. **임베딩 모델의 성능**
   - 가장 일반적으로 사용되는 Transformer 기반 모델들(BERT)은 입력 텍스트의 최대 길이가 약 512 토큰→ 따라서, 512 토큰을 기준으로 청크를 나누는 것이 일반적임
   - **제일 중요한 파트임.**
3. 사용자의 쿼리 길이와 복잡성
   - 짧고 구체적 VS 길고 복잡
4. If, 검색된 결과를 활용한다면
   - 검색, 질문 답변, 요약 혹은 다른 목적의 활용으로 토큰 제한이 고려되고, 이에 Chunk수를 제한해야함


### Cleansing (정제) 및 Normalization (정규화)

- Tokenization (토큰화) 
  -  자연어 코퍼스를 목적에 맞게 최소한의 의미 단위 (토큰)로 분절하는 작업
- 토큰화 작업 전/후에는 텍스트 데이터를 용도에 맞게 정제 및 정규화 하는 작업이 필요함
    - 정제
      - 갖고 잇는 코퍼스로부터 노이즈 데이터를 제거
    - 정규화
      - 표현 방법이 다른 단어를 통합하여 같은 단어로 만듦


- Cleansing
18토큰화 작업에 방해가 되는 부분을 필터링
토큰화 작업 이후에도 남아있는 노이즈들을 제거하기위해 지속적으로 이루어지는 전처리 과정
모든 노이즈를 완벽하게 제거하는 것은 불가능 è 일종의 합의점을 찾아야하는 작업

  -  토큰화 후 
     -  자연어처리 작업을 이해하는데 방해되는 특성(노이즈)를 검사하고 필터링

-  정규화
   -  여러 가지 정규화 기법이 존재

## Word Embedding 
- 중요한 개념임

- 임베딩
  - 자연어 처리 작업에서 특징 추출을 통해 자연어를 수치화 하는 과정 (자연어 è 벡터) 
  -  (예시) 학생 è [0, 1, 0, 1, 0, 0, …, 0, 1]
-  역할
    - 자연어의 의미적인 정보 함축
    - 자연어 간 유사도 계산

# Embedding의 역할
- 자연어 간 유사도 계산
  - 컴퓨터는여러개의특징차원으로구성된공간상에서사람처럼벡터의의미해석가능
  - 유사도 계산 방법을 활용하여 두 벡터(두 단어)간 유사도를 계산 가능
- 코사인 유사도
  - 두벡터간의코사인각도를이용하여두벡터의유사도를계산
  - -1~1사이의 값으로 표현 (1에 가까울수록 유사도 높음)
- 벡터 공간 시각화
  - 차원 축소 기법으로 저차원(2~3차원)으로 시각화 가능
  - 차원 축소 기법: t-SNE, PCA
