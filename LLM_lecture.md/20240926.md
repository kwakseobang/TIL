### Tokenization (Chunking)
- 일반적인 전처리 (Preprocessing) 과정
  - 데이터 수집 과정에서 얻어낸 코퍼스 데이터가 필요에 맞게 처리되지 않은 상태라면, 해당 데이터를 용도에 맞게 전처리하는 작업이 필요
- 원시 데이터(Raw data)로부터 유용한 데이터 문자열로 변환하는 간단한 프로세스
- 단락과 문장을 보다 쉽게 의미를 할당할 수 있는 더 작은 단위로 활용
**주어진 코퍼스 내 자연어 문장들을 “Token”이라는 최소 단위로 나누는 작업**
보통 띄어쓰기로 나누는 경우가 많음
- Token
   - 자연어 내에서 의미를 가지는 최소 단위



### How to tokenize (Chunk)

- Sentence Tokenization (Chunking)
  - 토큰의 단위가 문장 (Sentence)인 토큰화 방법
  - 문장부호 (“.” “!” “?” 등)를 기준으로 코퍼스를 분리
  - 예외 존재
    - 문장 중간에 . 등 문장 부호가 있을 때. 예외처리를 해줘야함 커스텀해서 만져줘야댐

- Word Tokenization (Chunking)
  - 보편적인 토큰화 방법
  - 특정 구분기호를 가지고 텍스트를 나누게 되며, 기본적으로는 **공백을 구분자로 사용**


### Tokenization (Chunking) 시 고려사항 in LLM
1. Context 의 길이
   -  기사, 책과 같은 긴 문서 VS 트윗이나 메시지와 같은 짧은 콘텐츠
   -  **중요한 파트**
2. **임베딩 모델의 성능**
   - 가장 일반적으로 사용되는 Transformer 기반 모델들(BERT)은 입력 텍스트의 최대 길이가 약 512 토큰→ 따라서, 512 토큰을 기준으로 청크를 나누는 것이 일반적임
   - **제일 중요한 파트임.**
3. 사용자의 쿼리 길이와 복잡성
   - 짧고 구체적 VS 길고 복잡
4. If, 검색된 결과를 활용한다면
   - 검색, 질문 답변, 요약 혹은 다른 목적의 활용으로 토큰 제한이 고려되고, 이에 Chunk수를 제한해야함

## Word Embedding 

- 임베딩
  - 자연어 처리 작업에서 특징 추출을 통해 자연어를 수치화 하는 과정 (자연어 è 벡터) 
  -  (예시) 학생 è [0, 1, 0, 1, 0, 0, …, 0, 1]
  - 특징 추출이란?
    - 데이터 별로 어떤 특징을 가지고 있는지 찾아내고, 이것을 벡터로 변환하는 작업

-  역할
    - 자연어의 의미적인 정보 함축
    - 자연어 간 유사도 계산

# Embedding의 역할
- 자연어 간 유사도 계산
  - 컴퓨터는 여러 개의 특징 차원으로 구성된 공간 상 에서 사람처럼 벡터의 의미 해석 가능
  - 유사도 계산 방법을 활용하여 두 벡터(두 단어)간 유사도를 계산 가능
  - 코사인 유사도
    - 두 벡터 간의 코사인 각도를 이용하여 두 벡터의 유사도를 계산
    - -1~1사이의 값으로 표현 (1에 가까울수록 유사도 높음)
  - 벡터 공간 시각화
    - 차원 축소 기법으로 저차원(2~3차원)으로 시각화 가능
    - 차원 축소 기법: t-SNE, PCA

### 단어의 출현 빈도 기반의 embedding 구축
- 단어의 출현 빈도에 기반한 대표적인 embedding 구축 방법
  - One-hot encoding
    - 말 그대로 자연어를 0과 1로 구별하는 인코딩 방법
    - 자연어 코퍼스 내 전체 단어 집합의 크기를 벡터의 차원으로 설정
    - 표현하고 싶은 단어의 인덱스의 값=1,나머지=0
    - 단점
      - 희소 문제
        - 대부분의 값들이 0인 희소 행렬로 구성
           - 단어가 늘어날 수록 표현의 효율성 저하
           - 단어 사전이 커질수록 공간 복잡도 증가
      - 단어의 유사도를 표현할 수 없음
  - TF-IDF 방법
    - 단어의 빈도(TF)와 역 문서 빈도(IDF)를 사용
    - 문서-단어 행렬 내 각 단어들의 중요한 정도에 가중치를 주는 방법
    - 문서의 유사도 계산, 검색 시스템에서 검색 결과의 중요도를 정하는 작업, 문서 내에서 특정 단어의 중요도를 구하는 작업 등에 효율적

### Sparse representation
- 벡터 또는 행렬 대부분이 0으로 저장되는 표현 방법 즉 희소 표현
- One-hot encoding
  - 표현하고자 하는 단어의 인덱스값만 1 나머지는 0으로 표기하는 벡터 표현 방법
- TF-IDF
  - 문서 내 단어의 빈도를 기반으로 가중치를 적용해 문서의 중요한 단어를 나타내는 희소 벡터.
   
- sparse vector
  - One-hot incoding에 의해 만들어지는 벡터
  
### Dense representation in Deep Learning-based Model
- 현재 사용하는 기술
- 밀집 표현
  - 벡터의 차원이 조밀 (dense)해졌다는 의미
  - 0과 1의 binary가 아닌 연속적인 실수값으로 인코딩 됨
    - 강아지 -> [0.12, 0.84, 0.11, 0.34, 0.45] (fixed size, dim=5)
  - 특징
    - 적은 차원으로 대상을 표현 가능
    - 더 큰 일반화 능력을 가짐
    - 의미이해&정보검색향상
