## Closure
- 스위프트에서 클로저는 일회용 함수를 작성할 수 있는 구문이다. 익명 함수라 불리기도 함.


## 트레일링 클로저
- **함수의 마지막 인자값이 클로저** 일 때, 이를 인자값 형식으로 작성하는 대신 함수의 뒤에 꼬리처럼 붙일 수 있는 방식.
- 이떄 인자 레이블은 생략
 


~~~ swift

func divide(a: Int, succes s: () -> Void ) -> Int{
    defer{ //함수의 마지막에 실행된다.
        s()
    }
    return 100/a 
}

divide(a: 100) { () in 
    print("연산이 성공했습니다")
    }

~~~

## @escaping

- @escaping
  - 인자값으로 전달된 클로저를 저장해 두었다가, 나중에 다른 곳에서도 실행 할수록 허용해주는 속성.
~~~ swift
func callback(fn: () -> Void ) {
    fn()
}
callback {
    print("Closure가 실행되었습니다.")
}
~~~
- 이 구문은 정상적으로 실행된다. 위 함수를 아래처럼 바꾸면 오류가 발생한다.

~~~ swift
func callback(fn: () -> Void ) {
   let f = fn
   f()
}
~~~
- 그 이유는 Non-escaping 파라미터인 'fn'은 오직 직접 호출하다는 것만 가능하다. 그 이유는? 클로저는 기본적으로 **Non-escaping** 의 성격을 가진다.  즉,  해당 클로저를   **1. 함수 내에서 2. 직접 실행을 위해서만 사용해야 한다.** 이 때문에 함수 내부라 할지라도 변수나 상수에 대입 불가하다.


- 코드를 작성하다보면 클로저를 변수나 상수에 대입하거나 중첨 함수 내부에서 사용해야 할 경우가 있다. 이때 사용되는 것이 @escaping 속성이다. 이 속성을 클로저 옆에 붙여주면, 해당 클로저는 **탈출이 가능한 인자값** 으로 설정된다.(여기서 말하는 탈출이란, 함수 내부 범위를 벗어나서 실행되는 것)

~~~ swift
func callback(fn: @escaping () -> Void ) {
  let f =  fn
  f()
}
callback {
    print("Closure가 실행되었습니다.")
}
~~~
- 인자값으로 전달되는 클로저의 기본 속성이 탈출불가하도록 설정된 이유는?
  - 가장 큰 이점은 컴파일러가 코드르르 최적화하는 과정에서의 성능향상이다.
  해당 클로저가 탈출 할수 없다는 것은 컴파일러가 더 이상 메모리 관리상의 지저분한 일들에 관여할 필요가 없다는 뜻이다.
  - 탈출 불가 클로저 내에서는 self키워드를 사용할 수 있다. 왜냐하면 이 클로저는 해당 함수가 끝나서 리턴되기 전에 호출될 것이 명확하기 때문이다. 따라서 클로저 내에서 self에 대한 약한 참조를 사용해야 할 필요가 없다.
