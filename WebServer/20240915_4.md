# 모델

##  모델과 3티어

- 모델을 담당하는 영역을 다시 세분화 시켜서 데이터베이스를 담당하는 영역과 비지니스 로직을 처리하는 영역으로 구분해서 '서비스(로직 처리) 계층'과 영속(데이터 처리) 계층으로 분리하기도 한다.
- View 계층
  - JSP
  - Servlet
- 서비스 계층
  - 로직 처리
- 영속 계층
  - 데이터 처리
## DTO(Data Transfer Object)
- 데이터를 교환할 때 여러 개의 데이터를 묶어서 하나의 객체로 전달하는 것.
- 대부분은 Java Beans 형태로 구성함
  - 생성자가 없거나 반드시 파라미터가 없는 생성자 함수를 가지는 형태
  - 속성은 private로 작성
  - getter/setter를 제공할 것
- 이 외에도 Serializable 인터페이스를 구현해야하는 등의 규칙이 존재.

## 서비스 객체
- DTO는 단순히 여러 개의 데이터를 묶어서 하나의 객체를 구성하는 용도로 사용하고 주로 서비스 객체 메소드들의 파라미터나 리턴 타입으로 사용된다.
- 기능(로직)들의 묶음, 프로그램이 구현해야 하는 기능들의 실제 처리를 담당
- 예를 들어 CRUD 기능들은 모두 서비스 객체에 모아서 구현된다.
- Todo라는 대상을 다룬다면 이를 담당하는 서비스 객체는 TodoService라는 이름으로 클래스를 이용해서 처리하고, 컨트롤러들은 TodoService의 객체를 이용해 작업을 처리한다.
- 웹 MVC 구조에서는 화면에 필요한 데이터를 처리하고자 컨트롤러는 서비스 객체의 힘을 빌려서 처리한다.
- 예를 들어 화면에서 목록 데이터들이 필요하다면 TodoService의 get-List()의 결과를 받아서 JSP까지 전달할고 이를 JSP에서 보여주는 방식

## JSP - EL(Expression Language)
- ${}는 EL의 표현식이다.
- EL을 이용한 출력
  - EL을 이용하는 경우에는 자동으로 getter를 호출하게 된다.

## JSTL(JavaSercer Pages Standard Tag Library)
- JSP에서 동작하는 새로운 태그들의 묶음
- JSTL을 이용하기 위해서는 라이브러리가 존재해야된다. 프로젝트 생서이에 존재하는 build.gradle 파일에 의존성 라이브러리를 추가하나다.

## JSP파일에서 JSTL 사용하기
- 사용하기 위해서는 파일 상단에 태그 관련 설정을 추가해야 한다.
  - <%@ taglib uri="http://java.sun.com/jsp/jstl/core" prefix="c" %>
- JSP에서는 <%@ %로 작성되는 코드를 지시자라고 한다,
- **<c:forEach>**
  - JSTL에서 가장 많이 사용하는 반복문 처리
  - 배열이나 리스트 등의 처리에 사용
  - 속성
    - var
      - 변수 이름
    - items
      - 컬렉션
    - begin/end
      - 반복의 시작/ 끝 값
- **<c:if>, <c:choose>**
  - 제어문
- **<c:set>**
  - 변수를 생성하고 사용
