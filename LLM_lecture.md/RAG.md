# RAG
- 외부 데이터에서 적절한 문서 검색 및 답변을 생성하여 LLM 모델의 정확성과 신뢰성을 확장시키는 방법

# RAG 기본 개념 및 구성요소
- 검색 (Retrieval)
  - 학습된 데이터 X → 외부 데이터를 활용하여 사용자 쿼리와 가장 관련있는 데이터를 검색하는 과정
- 생성 (Generation)
  - 이전 검색 단계에서 도출한 가장 관련있는 정보와 사용자 쿼리를 함께 LLM 모델에 전달하여 최종
결과물을 생성

- RAG 모형 출현
  - 회수 모델이 질문과 관련된 정보를 DB에서 탐색하여 모델에 함꼐 입력.

- How ? → 사용자의 질문(Query)과 DB에 존재하는 문서(Document)의 벡터 간 유사도 기반으로 관련 정보 추출

RAG의 세가지 기술 패러다임
- Naïve RAG
  - 인덱싱(Indexing)
    - 외부 지식 데이터를 가져와 검색이 가능하도록 준비하는 단계
    -  PDF, CSV, 다양한 문서들의 데이터를 불러와서 텍스트 전처리하여 Vector DB에 저장해놓은 단계 -> 즉, 외부데이터의 벡터화
  - 검색(Retrieve)
    - 사용자의 입력을 문서에 검색하여 정보를 찾아내는 단계
    - 사용자 입력과 외부 데이터의 벡터들을 기반으로 유사도를 산출 후 TOP-K개의 관련 정보를 추출
  - 생성(Generation):
    - 증강된 정보를 원래 질문
과 통합해 답변을 생성하는 단계

- Advanced RAG 
  - 검색 전 프로세스
    - 데이터 인덱싱을 최적화하여 콘텐츠의 품질을 높이는 것을 목표
    - 텍스트의 표준화와 일관성을 확보
    - 청크 크기 조정, 날짜와 목적 등에 대한 메타데이터를 추가하여 문서와 사용자 질문의 일치를 개선하기 위해 정렬시킴
  - 검색 후 프로세스
    - 외부 데이터와 사용자 질문이 문맥 창의 제한을 초과하거나 노이즈가 발생하기 때문에 사용됨
    - 재순위화(ReRanking) 과정을 거쳐 가장 관련성 높은 정보를 문맥의 주요 위
치로 이동시켜 중요 정보 강조, 프롬프트 압축 등을 활용
